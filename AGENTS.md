## Long-running task protocol

For this project:

1. Read `TASK_STATUS.md` before making changes.
2. Maintain `TASK_STATUS.md` as the source of truth.
3. After each meaningful milestone, update:
   - completed steps
   - changed files
   - current judgment
   - unresolved issues
   - next step
4. If context is compacted, resumed, forked, or uncertain, re-read `TASK_STATUS.md` before continuing.
5. Do not rely only on conversation history for the active plan.
6. Before large modifications, write the plan to `PLANS.md`.
